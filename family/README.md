# Family

**Official challenge:** [Family on Geektrust](https://www.geektrust.com/candidates/coding/detailed/family)

**Canonical spec (Geektrust):** [`../data/geektrust/problems/family.json`](../data/geektrust/problems/family.json).

**Geektrust catalog:** Problem code **`BD-PS1-1`** · slug `family`.


## Spec in this repo

The Geektrust app bundle does **not** embed the full `problemStatement` / samples for this slug. This folder only holds a pointer.

- Refresh JSON + PDFs: `node scripts/sync-geektrust-data.mjs`
- Read the full problem on [Geektrust](https://www.geektrust.com/candidates/coding/detailed/family)

- **Problem Set PDF (local):** [`../data/geektrust/pdfs/PS1.pdf`](../data/geektrust/pdfs/PS1.pdf) — **PS1**
- **PDF on Geektrust:** [open PDF](https://www.geektrust.com/api/pdf/open/PS1)

## Sample I/O (from PDF)

Pairs under [`sample_input/`](sample_input/) and [`sample_output/`](sample_output/) are generated by **`cd scripts && npm install && node extract-pdf-samples.mjs`** (re-run after Geektrust changes the PDF).

## Run

`./run.sh` or `npm install` then `npm start -- sample_input/input1.txt`.
